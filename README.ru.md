# ReadAllMessages

*На других языках: [English](README.md)*

Маленький, но удобный мод для **Schedule I**: добавляет кнопку **«Read All»** в приложение Messages на игровом телефоне. Один клик помечает все диалоги как прочитанные и убирает значок непрочитанных уведомлений. И всё — никаких изменений геймплея, никакой автоматизации, ничего лишнего.

Надоело пролистывать 30 сообщений от барыг только чтобы убрать точку уведомления? Это для тебя.

## Возможности

- Добавляет синюю кнопку **Read All** в приложение Messages — в шапке, рядом с заголовком *Messages*
- Помечает **все** диалоги прочитанными одним кликом
- Запасной вариант — клавиша **F8** делает то же самое из любого места
- Работает только локально — не трогает логику сейва и сетевое состояние, безопасен в мультиплеере
- Можно ставить и удалять в любой момент

## Требования

- **Schedule I** (Steam, обычная/основная ветка — IL2CPP)
- **[MelonLoader](https://melonwiki.xyz/#/?id=automated-installation) 0.7.0 или новее**

## Установка

1. Установи MelonLoader в Schedule I (запусти инсталлятор MelonLoader, укажи `Schedule I.exe`).
2. Запусти игру один раз, чтобы MelonLoader закончил настройку, и закрой её.
3. Скопируй `ReadAllMessages.dll` в папку `Mods` внутри папки игры:
   `...\steamapps\common\Schedule I\Mods\`
4. Запусти игру. При старте в консоли MelonLoader должен появиться `ReadAllMessages`.

> Возьми DLL из [последнего релиза](https://github.com/TheMangol/ReadAllMessages/releases/latest) или собери сам (см. ниже).

## Использование

1. Загрузи сейв.
2. Открой телефон и зайди в приложение **Messages**.
3. Нажми синюю кнопку **Read All** в шапке (рядом с заголовком *Messages*).
4. Все диалоги мгновенно станут прочитанными, значок уведомлений исчезнет.

Либо нажми **F8** в любой момент в игре — всё пометится прочитанным без открытия телефона.

## Сборка из исходников

Нужен только **.NET SDK 6.0** (или новее) и копия игры с установленным MelonLoader — для ссылок на сборки.

1. Запусти игру с MelonLoader хотя бы один раз и дождись главного меню. Это создаёт прокси-сборки в `Schedule I\MelonLoader\Il2CppAssemblies\`, которые нужны для сборки.
2. Открой `src/ReadAllMessages/ReadAllMessages.csproj` и проверь строку `<GamePath>` — путь к папке игры (по умолчанию `C:\Program Files (x86)\Steam\steamapps\common\Schedule I`).
3. Собирай из папки `src/ReadAllMessages`:

   ```bash
   dotnet build -c Il2Cpp   # обычная/основная ветка Steam (IL2CPP)
   dotnet build -c Mono     # ветка alternate (Mono, бета)
   ```

4. Если в `GamePath` есть папка `Mods`, готовый `ReadAllMessages.dll` скопируется туда автоматически. Иначе возьми его из `bin\Il2Cpp\` (или `bin\Mono\`) и положи в `Schedule I\Mods\` вручную.

## Возможные проблемы

- **Кнопки нет в Messages?** Посмотри консоль MelonLoader / `MelonLoader\Latest.log` на сообщения от `ReadAllMessages`. Мод ждёт появления приложения Messages до 2 минут после загрузки сейва. Клавиша F8 работает, даже если кнопку не удалось создать.
- **Не собирается — нет Il2CppAssemblies?** Запусти игру с MelonLoader один раз, дождись главного меню и собери снова.
- **Сборки называются иначе?** Открой `Schedule I\MelonLoader\` и проверь, что есть `net6\MelonLoader.dll` и `Il2CppAssemblies\Assembly-CSharp.dll`. Если пути отличаются — поправь их в `.csproj`.
- **Игра на ветке `alternate` (Mono)?** Релизная DLL собрана под основную ветку IL2CPP; для Mono собери из исходников с `-c Mono`.
- **Что-то сломалось после обновления игры?** Обновления могут менять внутренние имена классов. Открой issue и приложи `MelonLoader\Latest.log`.

## Замечания

- Мод вызывает только встроенную в игру функцию «пометить прочитанным» для каждого диалога — ровно то же, что происходит при ручном открытии диалога.
- Статус «прочитано» хранится в сейве как обычно. Никакие файлы вне игры не трогаются.
- Один исходник — две ветки: игровые типы используются как `ScheduleOne.*` (Mono) и `Il2CppScheduleOne.*` (IL2CPP), выбор происходит на этапе компиляции.
